# Mode 13 — /og-genesis

The offer inventor. Where /og-offer architects a product you already have, Genesis **generates the offerings themselves** — from what the business can actually deliver crossed with what the market is actively bleeding for. Read `winners-library.md` first; read `references/edge-engine.md` before naming anything.

## Inputs

1. **Capability inventory** — what the team/tech/founder can genuinely deliver at quality, today (from profile + a short confirm pass).
2. **Asset inventory** — certifications, IP, audience, data, relationships, infrastructure (Brand Guard's verified list in Mode O).
3. **Pain scan** — mine live signals, not vibes: competitor reviews (the 1–3 star ones are a goldmine), forum/Reddit complaints, job postings (a hire is a pain with a salary attached), search demand, sales-call objections on file.
4. **Constraints** — fulfillment capacity, compliance boundaries, cash-engine vs. equity-play intent.

## Process

```
Capability × Pain matrix → 10–20 raw candidates
  → score each: pain intensity · payer ability · delivery feasibility
                · differentiation ceiling · speed-to-cash   (1–5 each)
  → top 3–5 become Genesis Cards → human picks → hand to /og-offer
```

## Genesis Card (per surviving candidate)

```
name: <proprietary, ™-candidate — generic names are rejected at birth>
promise: <specific outcome, timeframe, for whom>
mechanism: <the NAMED "how" — own the method, not just the result>
persona seed: <who feels this pain at 3am — feeds /og-persona>
price hypothesis: <band + ladder position: entry/core/premium/recurring/fast-cash>
edge angle: <what makes this unlike the category — must pass the Only Test at concept stage>
fast-cash variant: <the 7-day-sellable version>
fulfillment sketch: <how it gets delivered without breaking>
kill criteria: <the evidence that would prove this offer wrong>
```

## Rules

- **No me-too offers.** If the concept can't pass the Only Test ("Only [brand]…" said honestly), it either gets a named twist or it dies here — before copy is ever written.
- Every mechanism gets a proprietary name. Unnamed methods are rented language.
- Speed-to-cash candidates get flagged for the cash-engine lane; moat candidates for the equity lane — never confuse the two in one card.
- Kill criteria are mandatory. An offer you can't disprove is a belief, not a product.
- Winning candidates flow: /og-genesis → /og-persona (build the buyer) → /og-offer (full architecture + Money Math) → /og-scripts.

## Produce (`artifacts/<slug>/genesis/`)

`candidate-matrix.md` (all raw candidates + scores), `genesis-cards.md` (the finalists), and the human's selection logged in STATE.
