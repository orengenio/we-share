# Mode 21 — /og-capture

The RFP Capture Machine. Doctrine: **the money is not in "responding to RFPs" — it's in a capture machine that finds right-fit opportunities, disqualifies bad ones fast, writes compliant winning proposals, and protects margin at the table.** Masters modeled: Plexico/GovWin (intel), Newman/Shipley (writing), Voss (negotiation) — see `references/masters-library.md`.

## The loop

```
FIND → QUALIFY → CAPTURE → WRITE → REVIEW → SUBMIT → NEGOTIATE → DELIVER → PAST PERFORMANCE
```

FIND/QUALIFY live in /og-opportunities (Tango MCP for live federal/SLED scans; A-grades hand off here). This mode owns CAPTURE onward.

## Capture plan (starts BEFORE the RFP drops)

Customer intel (mission, hot buttons, incumbent performance) · incumbent strength honestly assessed · teaming targets where set-aside or capability gaps exist · win strategy drafted pre-solicitation. A proposal written cold on drop day is already losing to whoever captured for six months.

## Bid/no-bid gate (GovWin discipline — enforced, not vibes)

p-win estimate · capacity to deliver at quality · margin after compliance overhead · every requirement answerable · set-aside eligibility per Brand Guard's verified list — **never chase or claim set-asides not held.** Bad RFPs drain time and kill margins; "no-bid" is a profitable decision and gets logged like a win.

## Proposal build (Shipley spine)

1. **Compliance matrix FIRST** — mandatory artifact: every shall/must/will → owning section → page. If the evaluator has to hunt for compliance, you lose points you earned.
2. **Evaluation criteria mapping** — write to the scoring rubric's weights, not to what's fun to say.
3. **Win themes** — this is the Edge Engine wearing a suit: every theme is an Only-Test claim with receipts (feature → benefit → PROOF). Themes ghost the competition ethically — highlight discriminators, never name-and-shame.
4. **Volumes** — technical solution · management approach · past performance · price strategy · risk reduction woven through all of them.
5. **Executive summary** — written last, read first, decision-maker language, themes up front.

## Color reviews (mapped to gates)

**Pink** — structure/storyboard vs. compliance matrix (before drafting deep). **Red** — scored dry-run against the actual evaluation criteria by someone who didn't write it; score it like the evaluator will. **Gold** — final compliance sweep + polish + Brand Guard/Edge pass. No submission without Red review logged in STATE.

## Negotiation layer (Voss)

Set the margin floor BEFORE the call — floors decided under pressure aren't floors. Then: tactical empathy · mirrors and labels · **calibrated questions** ("How are we supposed to deliver that customization inside that budget and still hit the performance requirements?") beats "we can't" every time · no-oriented questions to safe-close · anchor scope, control risk · **trade concessions, never give them** — every give gets a get (scope cut, timeline relief, payment terms). Floor breach or >$10k exposure → /og-board before agreeing. Battlecards for the table are built with /og-scripts.

## Deliver → past performance (the compounding asset)

Delivery excellence documented as it happens: metrics, references, CPARS-mindset evidence file. Past performance is the moat every future bid stands on — write the entry to `winners-library.md` and the past-performance file the week of delivery, not the night before the next bid.

## Produce (`artifacts/<slug>/capture/`)

`capture-plan.md` · `bid-no-bid-record.md` · `compliance-matrix.md` · `win-themes.md` · `volumes/` · `review-log.md` (Pink/Red/Gold) · `negotiation-battlecard.md` (floor, gives/gets, calibrated questions) · `past-performance-entry.md`.
