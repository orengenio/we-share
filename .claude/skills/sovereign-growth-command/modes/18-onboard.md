# Mode 18 — /og-onboard

The GHL client onboarding engine: signed → provisioned → configured → trained → live. This is the Mode C delivery SOP — repeatability here IS the margin. Instantiate the runbook per client; never onboard from memory.

## Day 0 — Provision

☐ Sub-account/location created at agency level; timezone, business info, branding set
☐ **Snapshot deployed** (see Snapshot Doctrine below) — or flagged net-new build via /og-ghl-sync
☐ Users + roles provisioned; client's outbound-approver named in profile
☐ Domain connected: DNS checklist (A/CNAME), SSL verified
☐ Email sending domain authenticated: DKIM / SPF / DMARC checklist — no sends before green
☐ Phone number purchased + **A2P 10DLC brand & campaign registration submitted** — hard dependency: SMS is blocked until carrier approval, and approval takes days, not minutes. Set that expectation in the kickoff, in writing. (Full rules: `references/compliance-rules.md`)
☐ Payment gateway connected (GHL payments or Stripe per profile); test transaction planned
☐ Calendars connected (Google/Outlook), availability + buffers set

## Day 1–3 — Configure

☐ /og-ghl-sync build executed (or snapshot customized: swap custom values, brand tokens, offer copy)
☐ Contacts imported — **consent-clean lists only**; unknown-provenance lists go to re-permission, never cold sends
☐ Forms, funnels, and /og-website wiring live; consent checkboxes verified writing to custom fields
☐ Workflows activated in dependency order: **opt-out kill-switch FIRST**, then capture, then nurture, then sales sequences
☐ Full-path test lead run (the /og-ghl-sync verification checklist): tags, stages, workflows, bump→upsell→downsell routing, opt-out kills everything

## Day 3–7 — Train & hand off

☐ Training walkthrough recorded (script generated by this mode): inbox, pipeline, calendar, mobile app, "the three things to never touch"
☐ One-page client SOP: where leads land, how fast to respond (speed-to-lead is theirs to own), how to ask for help
☐ Escalation path + approval rights confirmed on their side
☐ Go-live through the Launch Gate (modes/10-launch.md) — client launches get the same 11 points as internal ones

## Day 7 / 14 / 30 — Adoption cadence

Day 7: are they actually answering leads? Usage check, friction fixes. Day 14: first numbers read (directional only). Day 30: mini /og-optimize pass, Money Math vs. actuals, and the natural upsell trigger — capacity pain → next ladder rung.

## Snapshot Doctrine (the productization law)

Every completed /og-ghl-sync build gets **generalized into a reusable snapshot**: client-specific values stripped to custom values/variables, versioned, added to the snapshot library with a manifest (niche, funnel type, workflows included, A2P notes). The library compounds: deploy in minutes, deliver in days, same margin math as software. `winners-library.md` tracks which snapshot versions convert — snapshots evolve like everything else here.

## Portability clause (brand-enemy alignment)

The client owns their data and, per agreement, their sub-account access path. We keep clients with results — never with hostage mechanics. Offboarding steps documented in the runbook from day one.
