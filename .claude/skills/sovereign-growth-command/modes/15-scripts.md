# Mode 15 — /og-scripts

The script armory. Objection-fighting, pain-activating scripts for every conversation the business has — built FROM the persona files, edge-checked so they sound like nobody else, compliance-stamped where the channel demands it. Requires: at least one persona on file (run /og-persona first — scripts without personas are karaoke).

## The armory (generate what the engagement needs)

- **Cold openers** — call + DM + email variants per persona, opening with THEIR verbatim pain language, not product language.
- **Discovery call flow** — question ladder that surfaces the pain hierarchy in order, prices the status quo, and earns the pitch.
- **Objection battlecards** — one per objection in the persona stack: `acknowledge (their words) → reframe (the root fear, named) → proof (specific, on file) → close question`. No debate-club energy; Trusted Counselor tone.
- **VSL script** — hook → problem (voice-of-customer) → named mechanism → proof → offer stack → close, with the Edge move in the first 10 seconds.
- **Webinar close** — transition, stack, price reveal (Money Math honest), guarantee, Q&A objection sweep.
- **Buy-Lingual™ voice-AI call flows** — greeting (AI disclosure where required, baked in), qualification tree, objection branches, booking close, opt-out handling. Compliance stamps mandatory; multilingual variants are localizations, not translations.
- **Follow-up set** — no-show, ghosted, post-demo, post-purchase, win-back. Each references something specific from the prior touch.
- **Price conversation** — anchoring, tier walk, "most popular" framing (only where configured), payment-plan downsell language.

## Pain activation (the technique, named)

Activation ≠ agitation theater. The sequence: **surface it in their words → make it present-tense → price it (cost-of-inaction math from Money Math) → then, and only then, relieve it.** Pressure through truth. Banned: fake scarcity, manufactured countdowns, negging, "slots filling fast" fiction. If the urgency isn't real, the script finds real urgency (deadline, trigger event, compounding cost) or drops the urgency play.

## Script anatomy (every script carries this header)

```
persona: <codename> | context: <channel + moment> | pain activated: <which tier>
edge move: <the signature construction used — per voiceprint>
objection branches: <count> | CTA: <exact ask + destination> | GHL tag on outcome: <tag>
compliance: <stamp + risk rating for SMS/voice/DM channels>
edge score: <n/10 — ships at ≥7>
```

## Rules

- Every script passes the Edge Engine (`references/edge-engine.md`) AND Brand Guard. Generic scripts are returned for rewrite, not shipped with apologies.
- Outbound scripts enter the approval queue like every other send.
- Winning lines (measured, not felt) get written to `winners-library.md` and become voiceprint signature moves.

## Produce (`artifacts/<slug>/scripts/`)

One file per script type per persona + `battlecards.md` (the full objection matrix) + approval-queue entries for outbound sets.
